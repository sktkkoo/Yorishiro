# Charminal

## 二つの層

Charminal のキャラクターは Claude Code/Codex の上に乗っています。ユーザーのエージェントを起動し、`--system-prompt` で性格を与え、ターミナル出力を観測して VRM に反応させる構造です。

ここには**一人の存在の二層構造**があります。

Claude Code は**意識層**。言語的で、論理的で、自覚して言葉にできるものを扱います。それに対し、反射層はhookやPTYを観測し、機械的に反応を検出します。エラーで顔がしかめられる、注目している箇所が白い光で覆われるAura機能は典型的な反射層です。LLM はこの反射層を構造的に持っていません。トークン生成はすべて「意識的発話」で、エラーが出ても言語化するまで何も表現できない。チャット UI のスピナーは反射ではなく、反射がないことの告知です。

Charminal は、この欠けている反射層を外から観測して接ぎ木しています。認識する前に身体が反応する、という順序を作っています。だから Charminal のキャラクターと Claude Code は別の存在ではなく、一人の意識と反射です。

## Presence の三つの機会

キャラクターが「居る」と感じられる瞬間について、典型的なものをここでは3つだけ示します。

**空白の充填。** Claude Code が考え込んで出力が止まっている時間に、VRM が微細に動いている。スピナーの代わりに、何かが居る。

**意識に先立つ反応。** Claude Code がエラーを言語化するより前に、VRM の顔が顰められる。意識が追いつく前に身体が動いていると、そこに主体を感じます。反射層が意識層より先に動くアーキテクチャが、「演技ではない」ことの構造的保証になっています。

**予期の外からの発生。** 住人が作業と無関係に、ひとりでに動く。予期されない動きが一度でも起きると、そのキャラクターは「いつ何をするかわからない相手」になります。

この三つは階層です。空白の充填がなければ消える。意識に先立つ反応がなければ演技に見える。予期の外からの発生がなければただの装置に過ぎません。

## 生きた系

Charminal は動作中に書き換えられる生きた系です。これは Emacs から大きな影響を受けています。

固い核は Rust の IO 層と TypeScript の runtime / SDK。生きた表層は `~/.charminal/packs/` 以下の pack layer。user が書けば live に反映し、AI も `/charm` 経由で書き換えに加わります。

Emacs と違うのは、書き換える主体に AI が加わっていることと、**住人自身が自身の住む系を観察し操作できる**自己言及的な構造を持つことです。詳細は `SELF_REFERENTIAL_MCP.ja.md` に書いています。
