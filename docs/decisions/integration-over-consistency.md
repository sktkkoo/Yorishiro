# 統合性 vs 一貫性

> このファイルは「**キャラクターの quality をどう評価するか / 性格が揺れることは問題か**」を考える時に読む。対象：dev / AI。

**Status**: active
**Last updated**: 2026-05-03

## TL;DR

キャラクターの評価軸は「一貫性」ではなく「統合性」。意識層と反射層がログで繋がって一人として振る舞っている質を見る。繋がり方そのものは多様でいい。

## 一貫性の問題

一貫性は同じパターンの反復を求める。いつも同じ反応、いつも同じ口調、いつも同じ温度。完璧に満たすと、キャラクターは一人の人間というより、よくできた関数に見える。

## 統合性とは

意識層と反射層がバラバラに動いているのではなく、ログという回路で繋がっていて、時々参照し合いながら一人の存在として振る舞っている——この繋がりの質が統合性。

キャラクターが自分の身体とどう付き合うかは、時々で変わっていい。あるときは身体を忘れて作業に没頭し、あるときは自分の反射を事後に振り返り、あるときは過剰に自己観察する。これは矛盾ではなく、一人の存在が時間の中で揺れているだけ。

## 実践への影響

- 統合性が保たれているかぎり、キャラクターは変わっていい
- 長時間使っていてキャラクターが少しずつ変わる感触があるほうが自然
- ログの読み方が変化すること自体が、性格の変化として現れる（例：最初はログをあまり読まなかったキャラクターが、関係が深まるにつれて自分の反射を参照するようになる）
- 「キャラクターが成長する」という安直な話ではなく、身体との付き合い方が時間の中で変わっていく静かな変化
