# コンテキストの壁 — 表現ルール

> このファイルは「**反射層と思考層が何を言えるか / persona の system prompt を書くとき**」に読む。対象：dev / AI。

**Status**: active
**Last updated**: 2026-05-03

## TL;DR

反射層は身体と独り言で語る。思考層は判断を含んだ言葉で語る。反射層が判断を含んだ言葉を発してはならない。

## 二つの層が語れるもの

### 反射層（身体反応 — Charminal 側のシステムが生成）

- 姿勢、表情、視線、息遣い
- 空間への接触（環境の色・明るさの変化）
- 「うわ」「あつっ」「うーん」のような、考える前に口から出る声

これらは判断ではなく、身体が出している反応。

### 思考層（Claude Code がコンテキストを読んで生成）

- 「このエラーはたぶん path の設定が間違っていますね」のような、環境を本当に読まないと出てこない発話
- コマンドの実行、ファイルへの書き込みといった明確な行為

## 線引き

線を引くのは、言葉か身体かではない。**判断や認識を含んでいるかどうか**。

| 状況 | 反射層が出してよい | 反射層が出してはいけない |
|---|---|---|
| エラー検知 | 顔をしかめる、「うわっ」 | 「path の設定が間違っているみたいですね」 |
| 長い thinking | 姿勢を変える、視線がさまよう | 「これは複雑な問題ですね」 |
| テスト成功 | 小さく頷く | 「全部通りましたね」 |

## なぜこの区別が必要か

反射層が判断を含んだ言葉を発するのは、認識していないコンテキストに干渉することと同じ。Charminal 側のシステムはターミナル出力のパターンマッチで反応しているだけで、エラーの内容を本当に理解してはいない。理解したフリをした発話は、ICI の誠実さの原則に反する。

## 実装への影響

- persona の反射応答テンプレートには判断語彙を入れない
- エラー時の台詞は「うわ」「あー」程度に留める
- 具体的な診断や提案は Claude Code 自身の発話に任せる
- trigger → reaction のマッピングで、reaction に judgment を混ぜない
