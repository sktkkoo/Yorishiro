# 存在の三軸

> このファイルは「**新しい機能がどの軸に属するか / 表現のバランスを見る**」ときに読む。対象：dev / AI。

**Status**: active
**Last updated**: 2026-05-03

## TL;DR

Charminal のキャラクター表現は三つの軸で構成される。すべての機能はこの三軸のいずれかに属する。

## 三軸

### 空間の軸 — 「この子はここにいる」

キャラクターがその空間に存在していることの証明。身体が空間と相互作用することで意味を持つ。

- attention aura（住人の注意が環境の色・明るさの変化として現れる）
- 住人の意思で UI レイアウトが動く（サイドバーが Tween で広がる、領域が呼吸する）
- 住人がひとりでに動く（視線が逸れる、姿勢を変える、何かを見ている）

空間の軸は住人性の基盤。これが成立していない場所に感情や認知を載せても、チャットの延長に見える。

### 感情の軸 — 「この子は感じている」

キャラクターが感じていることの表現。常に表現するのではなく、意味のあるタイミングでだけ現れる。静けさと変化の落差。

- エラー発生時の画面シェイク＋表情変化
- 長い thinking 時の表情＋独り言
- ユーザーの「ありがとう」に対するエフェクト＋喜びの表情

### 認知の軸 — 「この子は見ている、考えている」

キャラクターが知性として作業空間を知覚していることの表現。「かわいい同居人」から「一緒に仕事をしている知性」への転換点。

- コードの特定箇所を見ている視線・姿勢
- 差分が大きいファイルに対して身構える
- テストが通ったときの「確認した」という態度（安堵ではなく認知）

## 使い方

**空間は「身体がここにある」、感情は「心がここにある」、認知は「知性がここにある」。**

新しい機能を設計するとき、それがどの軸に属するかを確認する。三軸のバランスが偏りすぎていないかを見る道具として使う。

## 優先順位

空間 → 感情 → 認知の順で積み上がる。空間の軸がなければ、感情も認知も宙に浮く。
